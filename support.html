Getting Started

Your GHL API key comes from a Private Integration you create inside GoHighLevel. Here's exactly where to find it:

  1. Log into your GoHighLevel account
  2. Go to Settings (the gear icon in the bottom-left sidebar)
  3. In the left menu, scroll down and click Integrations
  4. Click Private Integrations
  5. Click + Create New Integration
  6. Give it a name (e.g., "LeadCapture Pro") and enable at minimum these scopes: contacts.read, contacts.write, opportunities.read, opportunities.write, locations.read
  7. Click Save — the API key is shown once. Copy it immediately.
Heads up: GHL API keys are 150+ characters long. If yours looks short, you may have only copied part of it. Use Ctrl+A / Cmd+A in the key field to select all before copying.

Paste the key into the extension's Settings page → Step 2 → GHL API Key field.

Your Location ID identifies which GHL sub-account contacts are pushed into. There are two ways to find it:

Method 1 — Business Profile page:

  1. Log into GoHighLevel and open the sub-account you want to connect
  2. Go to Settings → Business Profile
  3. Scroll to the very bottom of the page
  4. Your Location ID is displayed there (looks like aBcDeFgHiJkLmN123456)

Method 2 — URL bar:

When inside a sub-account, look at your browser URL:

https://app.gohighlevel.com/v2/location/YOUR_LOCATION_ID/...

Copy the segment between /location/ and the next /.

Important: The API key and Location ID must belong to the same sub-account. Mismatching them causes a "403 Access Denied" error during the connection test.
Detection & How It Works

A few common causes and fixes:

1. Contact info is in an image, not text. Some sites display phone numbers as images so they can't be selected. Use the "Enter contact manually" button in the sidebar to type it in — it pushes to GHL the same way.

2. Strict Content Security Policy (CSP). Sites like Google, government pages, and banking apps sometimes restrict browser extensions. Click the toolbar icon to open the sidebar directly and use manual entry mode.

3. The confidence filter is hiding it. The extension automatically hides low-confidence detections when higher-quality data exists elsewhere on the page. Scroll through the full page to let dynamic content load, then re-check.

4. The page hasn't fully loaded. On slow or JavaScript-heavy sites, wait a moment for the page to finish rendering, then click the toolbar icon.

To confirm the extension is running, open DevTools (F12) → Console and look for messages starting with [LCP].

LeadCapture Pro checks for duplicates before creating anything. When a match is found you'll see the existing contact's name, when they were added, and their current tags.

You have two options:

  • Update — fills in any blank fields on the existing contact with new data from the page, and adds new tags additively. Nothing already in GHL gets overwritten.
  • Skip — dismisses the banner with no changes.

You can also click Check GHL before pushing to look up the contact manually without committing to any action.

Security & Privacy

Your API key is stored in Chrome's encrypted sync storage and is never:

  • Transmitted to our servers (we have none that receive it)
  • Included in the settings export file
  • Accessible to the web pages you visit
  • Logged to the browser console

The extension's architecture ensures only the background service worker — isolated from web page code — ever reads the API key. Content scripts that run inside webpages never see your credentials.

The only time your API key leaves your browser is when it's sent directly to GoHighLevel's API (services.leadconnectorhq.com) over an HTTPS connection.

For best security, create a dedicated GHL Private Integration with only the scopes LeadCapture Pro needs. You don't need to share your main GHL admin password.

No. We have no servers that receive or store contact data.

When you push a contact, the data travels directly from your browser to your GoHighLevel account. We are not in that data path. Detected contact data is held temporarily in your browser's local storage and is deleted automatically when you close the tab or navigate away.

For Agency Owners

White-labeling is built in. Here's the full process:

  1. Open the extension's Settings page
  2. In Step 1, enter your agency name, logo URL (a publicly hosted image), and brand color
  3. Go to Step 4 (Advanced) and click Download Settings JSON
  4. Send your clients the extension plus the JSON file
  5. Your client installs the extension, goes to Settings → Step 4, clicks Import Settings JSON, and selects your file
  6. Their sidebar immediately shows your agency branding

After import, your client enters only their own GHL API key in Step 2. API keys are never included in exports — each client's credentials stay private to them.

The extension is configured for one active GHL sub-account at a time. For multiple accounts, the easiest approach is:

  • Separate Chrome profiles — one per client or sub-account. Each profile has independent extension settings and credentials.
  • Manual switching — go to Settings, update the API key and Location ID, click "Test Connection" to confirm, then capture.

If you're distributing the extension to multiple clients, each client installs their own copy and enters their own credentials. Credentials are never shared between installations.

Yes. LeadCapture Pro connects via the GHL Private Integrations API, which works the same in standard Agency view, SaaS Mode, and white-labeled GHL instances.

What matters is that the sub-account has Private Integrations enabled and that you've created a key with the required scopes. SaaS Mode doesn't change this process.

Billing & Updates

No. LeadCapture Pro has no per-contact fees, no usage metering, and no subscription charges of its own. You pay once (or receive it as a bonus), and it works entirely through your existing GoHighLevel account.

GoHighLevel's standard Private Integration API is used — the same API your other GHL tools use — and there are no additional GHL charges for API usage at normal volumes.

Installed from the Chrome Web Store: Chrome updates extensions automatically within 24–48 hours of a new version being published. To force an immediate update, go to chrome://extensions, turn on Developer mode, and click Update.

Installed as an unpacked extension: Download the new version, replace the files in the extension folder, and click the ↺ refresh icon on the extension card in chrome://extensions.

Your settings are preserved across updates. You won't need to re-enter your API key or branding after updating.
